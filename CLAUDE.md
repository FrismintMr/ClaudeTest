## Versioned File Workflow
Every time a file is created or changed:
1. Save it as a **new versioned file** — never overwrite an existing file.
   - Use a version suffix: `script_v1.py`, `script_v2.py`, etc.
   - Or a timestamp suffix: `script_20260410.py`
2. `git add` the new file.
3. `git commit` with a clear message describing what changed.
4. `git push origin main` immediately.

This ensures every previous version of every file is always accessible in GitHub history.
